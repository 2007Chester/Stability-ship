# Замеры пресной воды (sounding) → тонны

## Можно ли переводить замеры в тонны?

**Да**, если есть **таблица калибровки** цистерны: зависимость **глубины замера (мм)** от **массы** или **объёма** (как в выгрузке **GHS** или в буклете по ёмкости). В приложении используется **линейная интерполяция** между узлами таблицы.

## Файл данных

Путь: **`stability_app/data/sounding_fresh.json`**

Формат:

```json
{
  "FRESH": [
    [0, 0],
    [5000, 50.0],
    [11000, 110.29]
  ]
}
```

- Ключ **`FRESH`** — **одна** таблица для **обоих** бортов (если цистерны симметричны и таблица общая).
- Или отдельно:
  - **`FRESH-PTS.P`** — левый танк (как в буклете),
  - **`FRESH-STB.S`** — правый.

Каждая строка: **`[замер_мм, масса_т]`**. Строки должны идти **по возрастанию** мм (при загрузке они сортируются). Минимум **две** точки.

Числа переносите из колонок **Snding / Weight METRIC TON** (или аналог) в GHS, либо из таблицы буклета по этой цистерне.

## Интерфейс

На вкладке **«Остойчивость»**, блок **«Пресная вода»**, раскройте **«Замер глубины (sounding) → тонны»**: введите мм, нажмите **«Подставить массу…»** — в поле массы танка подставится рассчитанное значение.

Пока файл пустой (`{}`), подсказка предлагает заполнить JSON вручную.

## Ограничения

- Не учитываются **дифферент и крен** — в GHS часто указано «No Trim, No Heel»; при большом дифференте таблица формально не совпадает с реальностью.
- **Плотность** уже заложена в колонку «Weight» в GHS для заданной жидкости; для пресной воды обычно **ρ ≈ 1,0 т/м³**.

## Связь с кодом

Модуль **`stability_app/sounding_tables.py`**: загрузка JSON, интерполяция `tons_from_sounding_mm`.
